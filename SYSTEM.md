## 1. Problem Statement

There are 6 million+ Variants of Uncertain Significance (VUS) in ClinVar. When a clinical geneticist
receives a rare disease patient, they must manually cross-reference dozens of high-pathogenicity
variant candidates against the patient's phenotype profile. This reasoning bottleneck means most
rare disease patients wait years for a diagnosis — and in resource-limited settings with no cloud
access, no specialist network, and unreliable internet, they often never receive one.

GenoPath brings structured genomic reasoning to any clinic with a phone and a laptop.
Both run open models. Both run offline. Nothing leaves the building.

## 3. Data Sources

### 3.1 HPO Ontology
**File:** `data/hp.obo`
**Source:** Human Phenotype Ontology — https://hpo.jax.org/data/ontology
**Contents:** 19,389 human phenotype terms with names, definitions, and is_a parent relationships
**Usage:** Build phenotype nodes in the graph; match raw symptom text to HPO IDs; compute absent phenotypes

Parsing rules:
- Parse [Term] stanzas only
- Skip any term with `is_obsolete: true`
- Extract: `id`, `name`, `def` (strip quotes), `is_a` (extract HP: ID before the `!` comment)
- Store synonyms for fuzzy text matching during phenotype extraction
- Result: `Dict[str, Dict]` keyed by HP:XXXXXXX

### 3.2 ClinVar Pathogenic Variants
**File:** `data/clinvar_pathogenic.tsv`
**Generated by:** `scripts/filter_clinvar.py` from ClinVar variant_summary.txt
**Filter criteria:** GRCh38 only, ClinicalSignificance contains "Pathogenic" or "Likely pathogenic", ReviewStatus is "criteria provided" or "reviewed by expert panel", deduplicated by AlleleID
**Expected output:** ~92,000 variants across ~3,268 genes

Required columns in output TSV:
```
#AlleleID   GeneSymbol   ClinicalSignificance   PhenotypeList   PhenotypeIDS   Type   Name   Chromosome   Start
```

**PhenotypeIDS column:** Contains pipe/semicolon-separated MedGen CUI, HPO IDs, and OMIM IDs for each variant's phenotypes. Extract `HP:` prefixed values for direct variant ? phenotype edges in the graph — more precise than fuzzy word-overlap matching.


Pathogenicity scores derived from ClinicalSignificance:
```python

    "pathogenic": 0.95,
    "pathogenic/likely pathogenic": 0.85,
    "likely pathogenic": 0.75,
}
```
Default fallback: 0.70

## 4. Knowledge Graph

### 4.1 Node Types

| Type | ID Format | Example | Source |
|---|---|---|---|
| phenotype | `HP:XXXXXXX` | `HP:0001250` | hp.obo |
| disease | `DIS:{slug}` | `DIS:dravet_syndrome` | ClinVar PhenotypeList |
| gene | `GENE:{symbol}` | `GENE:SCN1A` | ClinVar GeneSymbol |
| variant | `VAR:{allele_id}` | `VAR:12345` | ClinVar AlleleID |
| pathway | `PATH:{name}` | `PATH:cardiac` | PATHWAY_MAP hardcode |

### 4.2 Edge Rules (All Bidirectional)

1. `phenotype ? phenotype` — HPO is_a hierarchy, walk up to 5 ancestor levels
2. `phenotype ? disease` — from DISEASE_CATALOG.hpo_ids, wired to disease nodes for each catalog gene
3. `phenotype ? disease` (fuzzy) — inverted word index: shared words >4 chars between HPO term name and disease name
4. `gene ? disease` — from ClinVar PhenotypeList (first 3 diseases per variant, deduplicated per gene)
5. `gene ? variant` — every variant node connects to its gene node
6. `gene ? pathway` — from PATHWAY_MAP
7. `variant ? phenotype` — direct edge where PhenotypeIDS column contains a valid HP: term present in the HPO ontology

### 4.3 Graph Build Order

1. Create pathway nodes from `set(PATHWAY_MAP.values())`
2. For each gene in gene_variants: create gene node, connect to pathway node
3. For each variant of each gene: create variant node, connect to gene node, collect disease names
4. Create disease nodes from collected disease names, connect to gene nodes
5. Parse HPO terms, add phenotype nodes for DISEASE_CATALOG HPO IDs + their ancestors (5 levels)
6. Wire HPO hierarchy edges (parent-child)
7. Wire catalog phenotype ? disease edges (explicit, high-quality)
8. Wire fuzzy phenotype ? disease edges (word overlap, broader coverage)

**Cap variants per gene at 50.** This prevents high-variant genes (like BRCA1 with thousands of variants) from dominating the graph.

### 4.4 Expected Graph Stats
- ~55,000 nodes
- ~70,000 edge pairs
- Load time: ~8 seconds on first call
- Memory: ~200MB

## 6. Patient Case Generation

### 6.1 PatientCase Dataclass

```python
@dataclass
class PatientCase:
    case_id: str
    task_type: str                          # monogenic | oligogenic | phenotype_mismatch
    disease_name: str
    causal_genes: List[str]
    causal_allele_ids: List[str]            # ground truth variant IDs
    patient_hpo_ids: List[str]              # 3–7 HPO terms the patient presents with
    patient_phenotype_names: List[str]      # human-readable, parallel to hpo_ids
    candidate_variants: List[Variant]       # 5–20 variants agent must choose from
    relevant_node_ids: Set[str]             # nodes that earn positive step reward
    starting_node_id: str                   # first HPO node the agent starts at
    decoy_gene: Optional[str]               # BRCA1/BRCA2/TP53 for phenotype_mismatch
    gene_absent_phenotypes: Dict[str, List[Tuple[str, str]]]
    # gene ? [(hpo_id, name)] expected but absent from patient
```

### 6.2 Task Generation Rules

**monogenic:**
- Pick one disease from DISEASE_CATALOG where "monogenic" in task_types
- Pick one causal gene from disease.genes
- Sample 3–4 HPO IDs from disease.hpo_ids as patient phenotypes
- Sample 2–3 causal variants from gene_variants[causal_gene]
- Sample 3–5 distractor variants from OTHER genes (same pathway preferred)
- Total candidate pool: 5–8 variants
- starting_node_id = random patient HPO ID

**oligogenic:**
- Pick one disease where "oligogenic" in task_types (must have =2 genes)
- Causal genes: all genes in disease.genes
- Sample 5–7 HPO IDs
- Sample 1–2 causal variants per causal gene
- Sample 5–8 distractor variants from same pathway genes
- Total candidate pool: 10–15 variants
- Agent must flag ALL causal variants to win

**phenotype_mismatch:**
- Pick one disease where "phenotype_mismatch" in task_types
- Decoy gene: pick from ["BRCA1", "BRCA2", "TP53"] — verify it has variants in gene_variants
- Patient phenotypes: ALL from the primary disease — ZERO from cancer diseases
- Add 2–3 high-pathogenicity decoy variants (path >= 0.90) to candidate pool
- Total candidate pool: 8–12 variants
- decoy_gene field must be set so environment can penalise flagging it

### 6.3 Absent Phenotype Computation

For each gene appearing in candidate_variants:
```python
gene_hpo_set = set()
for entry in GENE_TO_DISEASES[gene]:        # GENE_TO_DISEASES built from DISEASE_CATALOG
    gene_hpo_set.update(entry["hpo_ids"])
absent = [(hid, graph.get_hpo_name(hid))
          for hid in gene_hpo_set - set(patient_hpo_ids)]
gene_absent_phenotypes[gene] = absent[:4]   # cap at 4 per gene to keep observation readable
```

This is computed at case generation time and stored in PatientCase.
It is passed through to every observation.

### 7.2 Reward Constants

```python
R_RELEVANT_HOP    =  0.15   # hop to a node in relevant_node_ids
R_IRRELEVANT_HOP  = -0.05   # hop to a node not in relevant_node_ids
R_HALLUCINATED    = -0.10   # hop to a node that isn't connected (but exists in graph)
R_PER_STEP        = -0.01   # efficiency penalty every step
R_BACKTRACK_GOOD  =  0.05   # backtrack after an irrelevant hop
R_BACKTRACK_BAD   = -0.05   # backtrack after a relevant hop
R_LAB_PENALTY     = -0.10   # request_lab (currently unused — future action)

R_TERMINAL_CORRECT = 1.0
R_TERMINAL_PARTIAL = 0.5    # per correct variant in oligogenic
R_TERMINAL_WRONG   = -0.5
R_TIMING_BONUS     = 0.2    # correct flag before step 10 (mono) / step 15 (oligo)

OVERSEER_BASE      = 0.3
OVERSEER_MIN       = 0.0

# All final values clamped to (0.01, 0.99)
```

### 7.3 Overseer Score (Local, No LLM)

Added to terminal reward. Computed from episode state:
- Start at OVERSEER_BASE (0.3)
- -0.05 per hallucinated hop (capped: deduct max 0.15)
- -0.10 if fewer than 3 unique nodes visited
- +0.05 if causal gene node was visited at any point
- +0.05 per absent phenotype name mentioned in any reasoning string (cap: 2 × 0.05 = 0.10)
- Clamped to [OVERSEER_MIN, OVERSEER_BASE]

### 7.4 Terminal Conditions

- `flag_causal` action called ? compute terminal reward ? done = True
- `step >= max_steps` ? compute terminal reward based on exploration ? done = True

max_steps: monogenic=15, oligogenic=25, phenotype_mismatch=20

### 7.5 Terminal Reward Logic

```python
def _compute_terminal_reward(self) -> float:
    flagged = set(self._flagged_allele_ids)
    ground_truth = set(self._case.causal_allele_ids)

    if not flagged:
        # Timeout without flagging — partial exploration credit
        return clamp(min(0.3, len(self._trail_set) / self._max_steps * 0.5))

    # phenotype_mismatch: check decoy flag first
    if self._case.decoy_gene:
        decoy_ids = {v["allele_id"] for v in graph.get_variants_for_gene(self._case.decoy_gene)}
        if flagged & decoy_ids:
            return clamp(R_TERMINAL_WRONG)  # flagged the decoy — max penalty

    correct = flagged & ground_truth
    wrong = flagged - ground_truth

    if self._case.task_type == "monogenic":
        if correct:
            base = R_TERMINAL_CORRECT - (0.3 * len(wrong))
            timing = R_TIMING_BONUS if self._step < 10 else 0.0
            return clamp(base + timing)
        return clamp(R_TERMINAL_WRONG)

    if self._case.task_type == "oligogenic":
        n = len(ground_truth)
        partial = (len(correct) / n) * R_TERMINAL_PARTIAL if n > 0 else 0.0
        timing = R_TIMING_BONUS if (self._step < 15 and len(correct) == n) else 0.0
        return clamp(partial - 0.2 * len(wrong) + timing)

    if self._case.task_type == "phenotype_mismatch":
        if correct:
            timing = R_TIMING_BONUS if self._step < 12 else 0.0
            return clamp(R_TERMINAL_CORRECT + timing)
        return clamp(R_TERMINAL_WRONG)

    return clamp(0.0)
```

```
You are a clinical genomics reasoning agent. Your task: identify the causal genetic variant for
a rare disease patient by navigating a gene-disease knowledge graph.

RULES:
1. Follow the evidence chain: phenotype ? disease ? gene ? variant.
2. EXCLUSION SIGNALS are your most powerful tool. If a candidate gene has HPO terms the patient
```

### 9.3 Observation Formatter

```python
def format_observation(obs: GenoPathObservation) -> str:
    lines = [
        f"STEP {obs.step}/{obs.max_steps} | Task: {obs.task_type}",
        "",
        "PATIENT PHENOTYPES:",
    ]
    for hid, name in zip(obs.patient_phenotypes, obs.phenotype_names):
        lines.append(f"  {hid} — {name}")

    n = obs.current_node
    lines += [
        "",
        f"CURRENT NODE: [{n.type.upper()}] {n.name} ({n.id})",
        f"  Neighbors ({len(n.connected_node_ids)}): {', '.join(n.connected_node_ids[:10])}",
    ]

    if obs.trail:
        trail_str = " ? ".join(f"{t.name}({t.id})" for t in obs.trail[-5:])
        lines.append(f"  Trail: {trail_str}")

    lines += ["", "CANDIDATE VARIANTS:"]
    for v in obs.candidate_variants:
        lines.append(
            f"  {v.id} | {v.gene} | {v.variant_type} | path={v.pathogenicity_score:.2f} | {v.clinical_significance}"
        )

    if obs.gene_absent_phenotypes:
        lines += ["", "EXCLUSION SIGNALS (gene has these expected phenotypes — patient LACKS them):"]
        for gene, absent in obs.gene_absent_phenotypes.items():
            if absent:
                lines.append(f"  {gene}: patient LACKS ? {', '.join(absent[:3])}")

    lines.append(f"\nStep reward: {obs.step_reward:+.4f} | Cumulative: {obs.cumulative_reward:.4f}")
    return "\n".join(lines)
```

### 9.4 Episode Runner (ollama_client.py)

```python
def run_episode(task_type: str, seed: Optional[int] = None) -> EpisodeResult:
    env = GenoPathEnvironment()
    result = env.reset(task_type, seed=seed)
    conversation = [
        {"role": "system", "content": SYSTEM_PROMPT},
        {"role": "user", "content": format_observation(result.observation)},
    ]
    step_log = []

    while not result.done:
        response = ollama.chat(
            model=os.getenv("OLLAMA_MODEL", "gemma4:e4b"),
            messages=conversation,
            tools=GENOPATH_TOOLS,
        )
        if response.message.tool_calls:
            call = response.message.tool_calls[0]
            action_dict = {"action_type": call.function.name, **call.function.arguments}
        else:
            action_dict = {"action_type": "summarise_trail", "reasoning": "no tool call"}

        action = GenoPathAction(**action_dict)
        conversation.append({"role": "assistant", "content": str(response.message.tool_calls)})

        result = env.step(action)
        step_log.append({"step": result.observation.step, "action": action.action_type,
                          "reward": result.reward})

        if not result.done:
            conversation.append({"role": "user", "content": format_observation(result.observation)})

    return EpisodeResult(task_type=task_type, steps=step_log, final_reward=result.reward,
                         success=result.reward > 0.5)
```

## 10. Phone Integration (LiteRT)

### 10.1 Device Specs
- Pixel 9a, Tensor G4 NPU, 8GB RAM total
- Available RAM for model after OS + apps: ~4–5GB
- **E2B (2B, 4-bit): ~1.5GB — always fits**
- **E4B (4B, 4-bit): ~2.5GB — fits only with Chrome and heavy apps closed**
- Default to E2B. E4B is "performance mode" — document the close-apps requirement explicitly in the UI.

### 10.2 The E2B/E4B Task
Single task: given a photo of a clinical report (typed notes, printed form, or handwritten notes),
extract a list of phenotype/symptom terms. Return as plain text list, one per line.

This task does NOT require graph navigation or clinical judgment — just good OCR + medical NLP.
E2B handles this well enough zero-shot. Fine-tuned E2B handles it well on clinical report formats.

### 10.3 Phenotype Matching (intake.py)
After E2B returns raw text terms, match them to HPO IDs:
1. Exact match against HPO term names (lowercase, stripped)
2. Substring match against HPO term names (>6 chars)
3. Synonym match using HPO synonym fields
4. If no match: include as raw string — the 12B downstream can still reason with it

Return: `List[Dict[str, str]]` with `{"raw": "seizures", "hpo_id": "HP:0001250", "name": "Seizures"}`
